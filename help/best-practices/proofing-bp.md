# 最佳实践 — 校对

## 什么是Adobe Workfront“最佳实践”？

最佳做法是代表有效、高效的行动方针的准则；易于您和贵公司用户采用；和可以在您的组织内成功复制。

在您查看这些建议时，请记住，一些Workfront最佳实践是普遍的，而其他一些则可能更具体地介绍该主题。 将这些最佳实践用作框架，以帮助指导Workfront系统的设置和使用。

## 导航此页面

在您浏览此页面时，首先您会找到该主题所有最佳实践的高级列表。 这样，您便可以审核推荐，而无需深入研究“原因”的详细信息。

“为什么这些最佳实践？” 高级列表后找到的区域，详细介绍了一些最佳实践以及它们被视为流程、工具等的原因，您应当考虑使用Workfront实例进行实施。

</br>
</br>

## Workfront中的校对最佳实践

* 请花时间创建校对工作流模板。

* 在Workfront设置中禁用“当对校样发表评论时从Workfront发送电子邮件”设置。

* 在Workfront中，对“打开文档校样的非收件人的角色”设置仅使用“只读”或“审阅人”。

* 调整校样后端设置，以便用户以12小时钟格式查看截止时间。

* 在系统设置中设置默认校样截止时间。

* 隐藏“不相关”校样决策选项。

* 请勿在校样设置中重新排序校样决策选项。

* 为校样角色和电子邮件警报设置用户默认值。

* 将校样创建者的校样角色设置为“审阅者”。

* 避免使用“审批者”校样角色。

* 避免显示所有活动校样电子邮件警报选项。

</br>
</br>

## 为什么是这些最佳实践？

**最佳实践**

请花时间创建校对工作流模板。

**原因如下**

模板不仅可以加快并简化校样创建和分配流程，而且还可以为类似类型的资产提供不同校样工作流程的一致性。 他们还可确保为每个校样收件人分配正确的校样角色和电子邮件警报，并设置截止日期。

</br>
</br>

**最佳实践**

在Workfront设置中禁用“当对校样发表评论时从Workfront发送电子邮件”设置。



**原因如下**

启用此设置（默认情况下为）后，用户可能会在校样上收到每个评论的多个电子邮件通知 — 一个来自校样功能，一个来自Workfront本身。 这些重复的通知会导致电子邮件通知中断和混淆，以及电子邮件收件箱（最终可能导致用户忽略他们收到的校样通知）。 这反过来又意味着错过最后期限。



**注意**:此设置位于Workfront主菜单>设置>电子邮件>审核和批准中。

</br>
</br>

**最佳实践**

在Workfront中，对“打开文档校样的非收件人的角色”设置仅使用“只读”或“审阅人”。



**原因如下**

此设置的其他选项都需要做出校样决策，这会引发校样工作流程脱轨。 通常，未添加到校样工作流的人员只需查看校样或进行评论，即无需实际批准校样，因此“只读”或“审阅人”选项是您的最佳选择。



**注意**:此设置位于Workfront主菜单>设置>审核和批准中。

</br>
</br>

**最佳实践**

调整校样后端设置，以便用户以12小时钟格式查看截止时间。



**原因如下**

对于希望以AM/PM格式查看校样截止时间/时间的用户，请在校样设置中选择F j、Y、gi:a选项。 对于使用12小时时钟的区域，这有助于明确截止时间。



**注意**:通过转到Workfront主菜单>校对>帐户设置>用户>并编辑每个用户的日期格式字段，可找到此设置。

</br>
</br>

**最佳实践**

在系统设置中设置默认校样截止时间。



**原因如下**

设置默认校样截止日期（上传日期+ x个工作日）后，如果校样创建者忘记添加截止日期，则Workfront会自动将此截止日期应用于上传的每个校样。



**注意**:通过转到Workfront主菜单>校对>帐户设置>设置>校样默认值>截止日期（+工作日）字段，可以找到此设置。

</br>
</br>


**最佳实践**

隐藏“不相关”校样决策选项。



**原因如下**

此决策选项通常会在批准者之间造成混淆，因为组织通常不会定义何时应使用“不相关”选项。 “不相关”选项通常指示证明与证明接收者无关，他们无需做出批准或拒绝的决定。 通过选择不相关，可以继续校样工作流。


大多数校样工作流中不需要“不相关”选项。

**注意**:通过转到Workfront主菜单>校对>帐户设置>决策，可找到此设置。

</br>
</br>

**最佳实践**

请勿在校样设置中重新排序校样决策选项。



**原因如下**

每个校样决策设置都有一个特定的值/权重，如果对该值/权重进行重新排序，可能会给您的校样配置带来混淆。 决策顺序和值/权重用作验证阶段激活触发器和报告中的触发器。



**注意**:通过转到Workfront主菜单>校对>帐户设置>决策，可找到此设置。

</br>
</br>

**最佳实践**

为校样角色和电子邮件警报设置用户默认值。



**原因如下**

在分配校样工作流、加快流程并促进校样工作流的一致性时，这些设置会自动填充。



**注意**:通过转到Workfront主菜单>校对>帐户设置>用户>并选择要为其设置默认值的用户，可找到用户默认设置。

</br>
</br>

**最佳实践**

将校样创建者的校样角色设置为“审阅者”。



**原因如下**

“审阅者”校样角色可确保校样创建者可以发表评论并访问他人留下的评论。 大多数情况下，校样创建者无需对他们上传的校样做出决策。 “审批者”、“审核者”和“审批者”、“作者”或“审核者”校样角色都需要做出决策。 如果为验证创建者分配了其中一个验证角色，但从未做出决策，则可能会对验证截止日期产生不利影响。

</br>
</br>

**最佳实践**

避免使用“审批者”校样角色。



**原因如下**

“审批者”校样角色不允许用户对此校样发表评论。 这可能导致用户拒绝校样，而且由于无法发表评论而没有任何说明。 请改用审核者和审批者校样角色，以便用户能够提供反馈。

</br>
</br>

**最佳实践**

避免显示所有活动校样电子邮件警报选项。

**原因如下**

当校样发生任何情况时，此选项会发送校样电子邮件通知 — 发表评论、发布回复、做出决策等。 收件人实际上看到的是校样活动。

对于校样所有者和创建者，决策电子邮件警报最适合于多阶段校样工作流，而最终决策则最适合于单阶段工作流。 通常，其他每个人都可以设置为“已禁用”，除非他们希望收到其他人做出评论或决策的通知（在这种情况下，其中一个摘要电子邮件选项可能最有效）。
