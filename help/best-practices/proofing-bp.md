# 最佳实践 — 校对

## 什么是Adobe Workfront“最佳实践”？

最佳实践是指导准则，它们代表了一个有效、高效的操作过程；您和您公司的用户可轻松采用；并且可在您的整个组织中成功复制。

在查看这些建议时，请牢记，某些Workfront最佳实践是通用的，而其他最佳实践可能更特定于此主题。 使用这些最佳实践作为框架，帮助指导您的Workfront系统设置和使用。

## 导航此页面

滚动浏览本页时，首先您会找到该主题所有最佳实践的高级列表。 这样，您就可以查看推荐，而无需深入了解“原因”的详细信息。

“为什么这些是最佳实践？” 区域位于高级列表之后，它提供了一些最佳实践的更多详细信息，以及为什么将它们视为流程、工具等，您应考虑使用Workfront实例进行实施。

</br>
</br>

## Workfront中的校对最佳实践

* 请花些时间创建验证工作流模板。

* 禁用Workfront设置中的“对验证进行评论时从Workfront发送电子邮件”设置。

* 对Workfront中的“打开文档验证的非收件人的角色”设置使用“只读”或“查看者”。

* 调整校样后端设置，以便用户以12小时时钟格式查看截止日期。

* 将默认验证截止日期建立为系统设置的一部分。

* 隐藏不相关的验证决策选项。

* 请勿在验证设置中重新排序验证决策选项。

* 设置验证角色和电子邮件警报的用户默认值。

* 将校对创建者的校对角色设置为查看者。

* 避免使用审批者验证角色。

* 避免使用所有活动验证电子邮件警报选项。

</br>
</br>

## 为什么有这些最佳实践？

**最佳实践**

请花些时间创建验证工作流模板。

**原因如下**

模板不仅可加快并简化校样创建和分配流程，而且还可跨相似类型资产的校样工作流提供一致性。 他们还确保为每个验证收件人分配了适当的验证角色和电子邮件警报，并且设置了截止日期。

</br>
</br>

**最佳实践**

禁用Workfront设置中的“对验证进行评论时从Workfront发送电子邮件”设置。



**原因如下**

启用此设置后（默认情况下启用），用户可能会收到校对上每个评论的多个电子邮件通知，一个来自校对功能，一个来自Workfront本身。 这些重复通知会导致电子邮件通知中断和混淆，以及完整的电子邮件收件箱，这最终可能会导致用户忽略他们收到的验证通知。 这进而可能意味着错过最后期限。



**注释**：此设置位于Workfront主菜单>设置>电子邮件>审阅和批准中。

</br>
</br>

**最佳实践**

对Workfront中的“打开文档验证的非收件人的角色”设置使用“只读”或“查看者”。



**原因如下**

此设置的其他选项都需要做出验证决策，这会引发验证工作流脱轨。 通常，未添加到验证工作流中的用户只需要查看验证或发表评论，而不是实际批准验证，因此“只读”或“查看者”选项是您的最佳选择。



**注释**：此设置位于Workfront主菜单>设置>审阅和批准中。

</br>
</br>

**最佳实践**

调整校样后端设置，以便用户以12小时时钟格式查看截止日期。



**原因如下**

对于希望以AM/PM格式查看验证截止时间/时间的用户，在验证设置中选择F j、Y、gi：a选项。 对于使用12小时制的地区，这有助于明确截止日期。



**注释**：通过转到Workfront主菜单>验证>帐户设置>用户>并编辑每个用户的日期格式字段可找到此设置。

</br>
</br>

**最佳实践**

将默认验证截止日期建立为系统设置的一部分。



**原因如下**

当设置默认验证截止日期（上传日期+ x个工作日）时，如果验证创建者忘记添加截止日期，Workfront会自动将此截止日期应用于上传的每个验证。



**注释**：通过转到Workfront主菜单>验证>帐户设置>设置>验证默认值>截止日期（+个工作日）字段可找到此设置。

</br>
</br>


**最佳实践**

隐藏不相关的验证决策选项。



**原因如下**

此决策选项通常会导致批准者之间混淆，因为组织通常未定义何时应使用“不相关”选项。 “不相关”选项通常指示验证与验证收件人无关，他们不需要做出批准或拒绝的决定。 通过选择不相关，这允许验证工作流继续。


在大多数验证工作流中，不需要“不相关”选项。

**注释**：通过转到Workfront主菜单>验证>帐户设置>决策可找到此设置。

</br>
</br>

**最佳实践**

请勿在验证设置中重新排序验证决策选项。



**原因如下**

每个验证决策设置都有一个特定的值/权重，如果重新排序，可能会给验证配置带来混淆。 决策顺序和值/权重用作验证阶段激活触发器和报告。



**注释**：通过转到Workfront主菜单>验证>帐户设置>决策可找到此设置。

</br>
</br>

**最佳实践**

设置验证角色和电子邮件警报的用户默认值。



**原因如下**

在分配验证工作流时，这些设置会自动填充，从而加快流程速度，并促进跨验证工作流的一致性。



**注释**：转到Workfront主菜单>验证>帐户设置>用户>并选择要为其设置默认值的用户，可找到用户默认设置。

</br>
</br>

**最佳实践**

将校对创建者的校对角色设置为查看者。



**原因如下**

查看者校样角色确保校样创建者可以发表评论并访问其他人留下的评论。 大多数情况下，校样创建者无需对已上传的校样做出决定。 “审批者”、“查看者和审批者”、“作者”或“审查者”验证角色都需要做出决定。 如果为校样创建者分配了其中一个校样角色，但从未做出决定，则可能会对校样截止日期产生不利影响。

</br>
</br>

**最佳实践**

避免使用审批者验证角色。



**原因如下**

审批者验证角色不允许用户对此验证进行评论。 这可能导致用户拒绝验证，且没有任何解释，因为他们无法进行评论。 请改用查看者和审批者验证角色，以便用户可以提供反馈。

</br>
</br>

**最佳实践**

避免使用所有活动验证电子邮件警报选项。

**原因如下**

只要验证出了问题 — 发表评论、发表回复、做出决定等，此选项就会发送验证电子邮件通知。 收件人在进行验证活动时基本上会看到该活动。

对于验证所有者和创建者，决策电子邮件警报最适合多阶段验证工作流，最终决策最适合单阶段工作流。 通常，其他所有人均可被设置为“已禁用”，除非他们想要收到有关其他人发表评论或做出决定的通知（在这种情况下，使用某个摘要电子邮件选项可能效果最佳）。
