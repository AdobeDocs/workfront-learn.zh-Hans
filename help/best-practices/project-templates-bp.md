# 最佳实践 — 项目模板

## 什么是Adobe Workfront“最佳实践”？

最佳实践是指导准则，它们代表了一个有效、高效的操作过程；您和您公司的用户可轻松采用；并且可在您的整个组织中成功复制。

在查看这些建议时，请牢记，某些Workfront最佳实践是通用的，而其他最佳实践可能更特定于此主题。 使用这些最佳实践作为框架，帮助指导您的Workfront系统设置和使用。

## 导航此页面

滚动浏览本页时，首先您会找到该主题所有最佳实践的高级列表。 这样，您就可以查看推荐，而无需深入了解“原因”的详细信息。

“为什么这些是最佳实践？” 区域位于高级列表之后，它提供了一些最佳实践的更多详细信息，以及为什么将它们视为流程、工具等，您应考虑使用Workfront实例进行实施。

</br>
</br>

## 项目模板最佳实践

* 创建项目时使用模板。

* 建立项目模板的命名约定。

* 建立一组选定的用户，这些用户可以创建和更新项目模板。

* 使用项目模板上的项目共享可自动授予对使用该模板创建的项目的访问权限。

* 将工作角色或团队分配给任务，而不是个人。

* 在项目模板中创建任务时，避免变得过于粒度。 将项目模板中的任务数限制为完成工作所需的任务数。

* 使用任务描述捕获任务的各个小步骤，而不是将其划分为多个任务。

* 确保模板任务包括任务持续时间、计划小时数和前置任务。

* 在模板上预配置项目详细信息并附加自定义表单。

* 定期查看和更新项目模板。

* 检查模板，确保模板在共享之前具有所有必需的信息，以便其他人开始使用。

* 在模板上调整“计划自”选项时，复查并更新任务限制。

* 检查模板上的项目团队，并删除不会与项目关联的用户。

</br>
</br>

## 为什么有这些最佳实践？

**最佳实践**

创建项目时使用模板。

**原因如下**

项目模板消除了项目经理（以及创建项目的其他人）对项目应包含哪些任务、如何构建时间线等的猜测。 模板是加速项目创建的最有效方法。

重要的是，模板提供了类型相似的项目之间的一致性，以便人员、流程和数据点每次都以相同的方式进行详细介绍。 即使是快速完成（一两天）和最小任务的项目也可以从使用项目模板创建的项目中受益。

跨项目的一致性可产生更准确的数据，这对于您的团队、部门以及整个组织做出决策至关重要。

</br>
</br>

**最佳实践**

建立项目模板的命名约定。

**原因如下**

一致的命名使模板更容易查找。 当多个团队或部门之间存在类似名称的模板时，它还可以帮助项目经理和其他创建项目的人选择正确的模板。

</br>
</br>

**最佳实践**

建立一组选定的用户，这些用户可以创建和更新项目模板。

**原因如下**

拥有构建良好、一致的项目模板是良好工作管理和准确报告的关键。 限制可编辑模板的用户数量，以避免意外更改或未经批准的更改。

</br>
</br>

**最佳实践**

使用项目模板上的项目共享可自动授予对使用该模板创建的项目的访问权限。

**原因如下**

特定项目的访问权限是通过项目本身授予的。 如果同一组人员始终需要访问使用特定模板创建的项目，请将其添加到模板上的项目共享选项下。 不仅您可以在项目创建后立即控制对项目的访问权限，而且如果未来需要更改权限，这还可以简化可扩展性工作。

**注释**：模板共享授予对模板本身的访问权限。 用户必须至少具有查看权限，才能使用模板制作项目。

</br>
</br>

**最佳实践**

将工作角色或团队分配给任务，而不是个人。

**原因如下**

当单个用户更改职位或离开组织时，您必须手动更新包含该人员的项目模板。 系统或组管理员或项目经理需要花费一些时间。

如果您在模板上使用工作角色或团队，人员配备更改将不会对您的项目模板产生直接影响，因为任何分配了该工作角色或该团队的任何人都可能被分配了该工作。 这有助于确保工作不会滑过裂缝。 工作角色分配还可以使将工作分配给个人用户变得更轻松，因为Workfront可以向您显示已分配该工作角色的人员列表。

此外，Workfront的资源计划工具还使用工作角色来帮助您计算所需资源和规划未来工作。

</br>
</br>

**最佳实践**

在项目模板中创建任务时，避免变得过于粒度。 将项目模板中的任务数限制为完成工作所需的任务数。

**原因如下**

过于复杂的项目模板会给用户（项目经理、资源经理、团队成员等）带来不良体验。 任务过多使项目时间表难以管理，因为任务截止日期重叠，并且多个任务被分配到相同的工作角色或个人。


</br>
</br>

**最佳实践**

使用任务描述捕获任务的各个小步骤，而不是将其划分为多个任务。

**原因如下**

如果一行中的多个任务被分配给同一个工作角色/个人，则表示这些任务可以合并。 分配给用户的任务过多，可能会让他们觉得还有更多工作要完成，这可能会影响Workfront的采用。

</br>
</br>

**最佳实践**

确保模板任务包括任务持续时间、计划小时数和前置任务。

**原因如下**

持续时间、计划小时数和前置任务这三样东西是项目时间表的构建基块。 这些是了解工作需要多长时间以及需要何时完成的关键。 Workfront的资源管理工具持续时间、计划小时数以及工作角色分配，用于计算资源容量、可用性等。

如果您不确定如何首次估计持续时间或计划小时数，请与项目团队合作定义一些初始估计。 使用模板后，再次与项目团队会面以确定可以在何处进行更改以使模板更准确。 如果用户在Workfront中记录时间，您可以将项目的计划小时数与实际小时数进行比较，以查看需要调整的位置。


</br>
</br>

**最佳实践**

在模板上预配置项目详细信息并附加自定义表单。

**原因如下**

确保在项目模板上填写了所有项目的标准信息。 这不仅有助于加快项目创建速度，还可以确保存在必要的信息并在各个项目中保持一致。

使用模板将请求转化为项目时，附加与请求自定义表单匹配的项目自定义表单，以提取提交的信息。

</br>
</br>

**最佳实践**

定期查看和更新项目模板。

**原因如下**

随着流程和团队的变更，应更新项目模板。 建立常规节奏（如每季度），以检查并查看哪些模板未积极使用。 您可以停用这些模板，因为它们仍在Workfront中，但不会显示在模板选择列表中。

</br>
</br>

**最佳实践**

检查模板，确保模板在共享之前具有所有必需的信息，以便其他人开始使用。


**原因如下**

由于模板将反复用于制作项目，因此您需要确保正确且完全地设置所有内容。 这有助于实现所有项目的一致性和用于报告的准确数据。

除了任务设置（如持续时间和计划小时数）之外，在共享模板之前还需要查看以下事项：

* 时间表开始日期设置
* 任务限制
* 项目所有者、发起人、组和公司
* Portfolio和计划
* 里程碑路径和步骤
* 审批流程
* 确保为项目分配任务的用户拥有项目的Contribute访问权限
* 资源池
* 提醒通知
* 计划
* 货币设置（如果适用）
* 附加标准文档
* 附上必要的海关表格
* 检查项目团队，确保没有额外分派人员

</br>
</br>

**最佳实践**

在模板上调整“计划自”选项时，复查并更新任务限制。

**原因如下**

在项目中混合使用不同的任务限制可能会导致计划日期计算意外和混乱。 例如，为“时间表开始日期”选项选择“开始日期”时，该项目中创建的任何任务都将默认分配为“尽可能早”任务限制。 如果您稍后将“调度自”选项切换到“完成日期”，则默认情况下，创建的任何任务都具有“尽可能晚于”任务限制。 无意中将任务与每个限制条件混合在一起可能会导致项目时间轴中的计划日期混乱。


</br>
</br>

**最佳实践**

检查模板上的项目团队，并删除不会使用该项目的用户。

**原因如下**

从现有项目创建模板时，它会带来项目中被分配任务/问题的人员。 而且，在使用模板时，您可以删除之前被分配过工作的人，或更改您在模板上自己完成的工作。

所有这些用户都将作为项目团队的一部分列在项目的“人员”和“计划”部分。 因此，它们将传播到使用该模板创建的所有项目。 这可能会导致用户混淆，因为作为项目团队的一部分，他们收到有关项目活动的通知，在“我参与的项目”列表中看到项目，并获得对项目及其任务、问题和文档的权限。
