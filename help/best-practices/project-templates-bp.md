# 最佳实践 — 项目模板

## 什么是Adobe Workfront“最佳实践”？

最佳做法是代表有效、高效的行动方针的准则；易于您和贵公司用户采用；和可以在您的组织内成功复制。

在您查看这些建议时，请记住，一些Workfront最佳实践是普遍的，而其他一些则可能更具体地介绍该主题。 将这些最佳实践用作框架，以帮助指导Workfront系统的设置和使用。

## 导航此页面

在您浏览此页面时，首先您会找到该主题所有最佳实践的高级列表。 这样，您便可以审核推荐，而无需深入研究“原因”的详细信息。

“为什么这些最佳实践？” 高级列表后找到的区域，详细介绍了一些最佳实践以及它们被视为流程、工具等的原因，您应当考虑使用Workfront实例进行实施。

</br>

## 项目模板最佳实践

* 在创建项目时使用模板。

* 为项目模板建立命名约定。

* 建立一组可创建和更新项目模板的选定用户。

* 对项目模板使用项目共享可自动授予对使用该模板创建的项目的访问权限。

* 将工作角色或团队分配给任务，而不是个人。

* 在项目模板中创建任务时，请避免过于精细。 将项目模板中的任务数限制为完成工作所需的任务数。

* 使用任务描述捕获任务的小步骤，而不是将其划分为多个任务。

* 确保模板任务包括任务持续时间、计划时间和前置任务。

* 预配置项目详细信息，并在模板上附加自定义表单。

* 定期查看和更新项目模板。

* 检查模板，确保模板在您共享之前具有所有必需的信息，其他模板则开始使用。

* 在模板中调整“计划自”选项时，请检查并更新任务约束。

* 在模板上检查项目团队，并删除不会与项目关联的用户。

</br>

## 为什么是这些最佳实践？

**最佳实践**

在创建项目时使用模板。

**原因如下**

项目模板消除了项目经理（以及其他创建项目的人）对项目应包含哪些任务、如何构建时间轴等的猜测。 模板是加快项目创建速度的最有效方法。

重要的是，模板可以提供相似类型项目之间的一致性，以便人员、流程和数据点每次都以相同的方式进行详细描述。 使用项目模板创建项目时，即使具有快速变通（一两天）和最少任务的项目也可能会受益。

跨项目的这种一致性可以生成更准确的数据，在为团队、部门和整个组织做出决策时，这些数据至关重要。

</br>
</br>

**最佳实践**

为项目模板建立命名约定。

**原因如下**

一致的命名使模板更易于查找。 它还有助于项目经理和其他创建项目的人员在多个团队或部门中具有类似名称的模板时，选择正确的模板。

</br>
</br>

**最佳实践**

建立一组可创建和更新项目模板的选定用户。

**原因如下**

构建良好、一致的项目模板是良好工作管理和准确报告的关键。 限制可编辑模板的用户数量，以避免意外或未批准的更改。

</br>
</br>

**最佳实践**

对项目模板使用项目共享可自动授予对使用该模板创建的项目的访问权限。

**原因如下**

通过项目本身授予对特定项目的访问权限。 如果同一组人员始终需要访问使用特定模板创建的项目，请在模板的项目共享选项下添加这些项目。 您不仅可以在项目创建后立即控制对项目的访问，还可以在未来需要更改权限时简化可扩展性工作。

**注意**:模板共享授予对模板本身的访问权限。 用户必须至少具有“查看”权限才能使用该模板制作项目。

</br>
</br>

**最佳实践**

将工作角色或团队分配给任务，而不是个人。

**原因如下**

当单个用户更改职位或离开组织时，您必须手动更新包含该人员的项目模板。 对于系统或组管理员或项目经理，这需要一些时间。

如果您在模板上使用职务角色或团队，则人员配置更改不会对项目模板产生直接影响，因为分配了该职务角色或该团队的任何人都可以分配该工作。 这有助于确保工作不会漏过裂缝。 职务角色分配还可以更轻松地将工作分配给个人用户，因为Workfront可以向您显示分配了该职务角色的人员列表。

此外，Workfront的资源规划工具还使用作业角色来帮助您计算所需资源并规划将来的工作。

</br>
</br>

**最佳实践**

在项目模板中创建任务时，请避免过于精细。 将项目模板中的任务数限制为完成工作所需的任务数。

**原因如下**

过于复杂的项目模板会给用户带来不良体验 — 项目经理、资源经理、团队成员等。 任务过多，使得项目时间轴难以管理，任务期限重叠，并且分配给相同作业角色或个人的多个任务。


</br>
</br>

**最佳实践**

使用任务描述捕获任务的小步骤，而不是将其划分为多个任务。

**原因如下**

如果一行中的多个任务被分配给同一作业角色/个人，则表示这些任务可以合并。 分配给用户的任务过多可能会让他们感觉有更多工作要完成，这可能会影响Workfront的采用。

</br>
</br>

**最佳实践**

确保模板任务包括任务持续时间、计划时间和前置任务。

**原因如下**

这三项内容（持续时间、计划时间和前置任务）是项目时间轴的构建基块。 这些是了解工作需要多长时间以及何时需要完成的关键。 Workfront的资源管理工具持续时间和计划时间，以及职务角色分配，以计算资源能力、可用性等。

如果您不确定如何首次估计持续时间或计划小时数，请与项目团队合作以定义一些初始估计。 使用模板后，再次与项目团队会面，以确定可以在何处进行更改以使模板更准确。 如果用户在Workfront中记录时间，您可以将项目的计划时间与实际小时数进行比较，以了解需要在何处进行调整。


</br>
</br>

**最佳实践**

预配置项目详细信息，并在模板上附加自定义表单。

**原因如下**

确保在项目模板中填写所有项目的标准信息。 这不仅有助于加快项目创建速度，还可确保存在必要的信息，并确保信息在各个项目中保持一致。

在使用模板将请求转换为项目时，附加与请求自定义表单匹配的项目自定义表单，以提取已提交的信息。

</br>
</br>

**最佳实践**

定期查看和更新项目模板。

**原因如下**

随着流程和团队的变化，项目模板也应该更新。 建立定期频度（如季度），以检查和查看哪些模板未被积极使用。 您可以取消激活它们，以便它们仍在Workfront中，但不会显示在模板选择列表中。

</br>
</br>

**最佳实践**

检查模板，确保模板在您共享之前具有所有必需的信息，其他模板则开始使用。


**原因如下**

由于模板将一次又一次地用于制作项目，因此您需要确保所有内容设置正确且完整。 这可以实现所有项目的一致性，并准确地提供报告数据。

除了任务设置（如持续时间和计划时间）之外，共享模板之前要查看的一些内容包括：

* 计划来源设置
* 任务约束
* 项目所有者、赞助商、组和公司
* Portfolio和计划
* 里程碑路径和步骤
* 审批流程
* 确保在项目上分配任务的用户具有对项目的Contribute访问权限
* 资源池
* 提醒通知
* 计划
* 货币设置（如果适用）
* 附加标准文档
* 附具必要的海关表格
* 检查项目团队以确保没有分配额外的人员

</br>
</br>

**最佳实践**

在模板中调整“计划自”选项时，请检查并更新任务约束。

**原因如下**

在项目中混合使用不同的任务约束可能会导致意外和混乱的计划日期计算。 例如，在为“计划自”选项选择“开始日期”时，默认情况下，在该项目中创建的任何任务都将分配“尽快”任务约束。 如果稍后将“计划自”选项切换到“完成日期”，则创建的任何任务都默认具有“尽可能晚”任务约束。 无意地将任务与每个限制混合在一起，可能会导致项目时间轴中的计划日期混乱。


</br>
</br>

**最佳实践**

在模板上检查项目团队，并删除不会处理项目的用户。

**原因如下**

当您从现有项目创建模板时，模板会覆盖为项目分配任务/问题的人员。 在您使用模板时，您可以删除之前已分配工作的人员，也可以更改您在模板上所做的分配。

所有这些用户都将作为项目团队的一部分列在项目的人员和计划部分中。 因此，它们将被传播到从该模板创建的所有项目。 这可能会给用户造成混淆，因为作为项目团队的一部分，他们会收到有关项目活动的通知，会在“项目我在列表”中查看项目，并获得项目及其任务、问题和文档的权限。
