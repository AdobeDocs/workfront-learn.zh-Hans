# 最佳实践 — 自定义表单

## 什么是Adobe Workfront“最佳实践”？

最佳实践是指导准则，它们代表了一个有效、高效的操作过程；您和您公司的用户可轻松采用；并且可在您的整个组织中成功复制。

在查看这些建议时，请牢记，某些Workfront最佳实践是通用的，而其他最佳实践可能更特定于此主题。 使用这些最佳实践作为框架，帮助指导您的Workfront系统设置和使用。

## 导航此页面

滚动浏览本页时，首先您会找到该主题所有最佳实践的高级列表。 这样，您就可以查看推荐，而无需深入了解“原因”的详细信息。

“为什么这些是最佳实践？” 区域位于高级列表之后，它提供了一些最佳实践的更多详细信息，以及为什么将它们视为流程、工具等，您应考虑使用Workfront实例进行实施。

</br>

## 自定义表单最佳实践

* 在Adobe Workfront中创建自定义表单之前，在白板或纸张上绘制该表单的图表。

* 如果使用该模板制作的项目始终需要附加某些自定义表单，请将自定义表单附加到项目模板。 您甚至可以填写很少更改或需要特定信息的字段。

* 创建自定义表单以跟踪有关您作为公司添加到Workfront系统的外部客户或供应商的信息。

* 创建包含常用字段和已计算字段的单个“常规”或“通用”自定义表单，而不是将单个字段附加到多个表单上。 这有助于您以一致的方式收集企业范围的信息。

* 将分区界限添加到自定义表单，使其保持井然有序，易于理解。

* 缩短自定义表单的长度，以确保表单已完全填写，并且您能够获得所需的所有信息。

* 使用预定义的自定义字段选项（如单选按钮和下拉菜单），通过要求用户从特定选项中进行选择来限制回答选项。

* 确保字段标签措辞明确且具有描述性。

* 在自定义字段的说明字段中添加信息，以指示填写该字段的人员必须输入哪些信息。

* 以特定方式填写其他字段时，使用显示逻辑显示所需字段。 在自定义表单中使用跳过逻辑可隐藏与请求类型无关的字段。

* 通过从其他表单中提取计算信息，减少自定义表单上所需的计算数量。

* 在为自定义表单创建新字段之前，请检查字段库，以查看字段是否已存在。

* 使用必填字段确保捕获到对您的流程或组织至关重要的信息。

* 在自定义表单中更改字段名称时请务必谨慎，因为这可能会影响调用该字段的计算字段。

* 作为常规Workfront系统维护的一部分，定期查看自定义表单和字段，例如每季度一次。

* 停用不需要的自定义表单，而不是删除它们。

* 在分区界限下隐藏自定义表单上不再需要的自定义字段。 然后，使此部分仅对系统管理员可见。

* 限制可以在Workfront实例中创建自定义表单的人员。

## 为什么有这些最佳实践？

**最佳实践**

在Adobe Workfront中创建自定义表单之前，在白板或纸张上绘制该表单的图表。

**原因如下**

确保您有字段用于所有必填信息，并确保这些字段的排列方式便于用户填写表单。 为表单绘制图表还可以帮助您确定是否可以使用显示逻辑隐藏/显示某些字段。

</br>
</br>

**最佳实践**

如果使用该模板制作的项目始终需要附加某些自定义表单，请将自定义表单附加到项目模板。 您甚至可以填写很少更改或需要特定信息的字段。

**原因如下**

这样，表单便已附加且部分信息已填写，这有助于加快项目创建并确保在所有适用的项目中正确完整地填写这些自定义字段。

</br>
</br>

**最佳实践**

创建自定义表单以跟踪有关您作为公司添加到Workfront系统的外部客户或供应商的信息。

**原因如下**

使用自定义表单跟踪地址、主要联系人的姓名等，以便在Workfront中轻松访问。 您还可以将此自定义表单信息提取到报表中。

</br>
</br>

**最佳实践**

创建包含常用字段和已计算字段的单个“常规”或“通用”自定义表单，而不是将单个字段附加到多个表单上。 这有助于您以一致的方式收集企业范围的信息。

**原因如下**

“常规”表单使维护这些字段变得更加容易，因为它们都位于一个位置。 您将能够更新单个表单，而不是让这些单独的字段位于多个不同的表单上，然后您必须单独更新。

</br>
</br>

**最佳实践**

将分区界限添加到自定义表单，使其保持井然有序，易于理解。

**原因如下**

将相关信息分组到多个部分可帮助用户导航表单。

</br>
</br>

**最佳实践**

缩短自定义表单的长度，以确保表单已完全填写，并且您能够获得所需的所有信息。

**原因如下**

长表格可能会令用户望而却步，并且通常会导致表格无法完全填写。 这进而导致分配信息不完整，报告的数据不准确。

如果自定义表单中有许多字段，请尽可能将相关字段并排放置，这样用户的滚动次数就会减少。 您还可以使用跳过逻辑来隐藏不需要填写的字段，或使用显示逻辑来显示特定字段。

</br>
</br>

**最佳实践**

使用预定义的自定义字段选项（如单选按钮和下拉菜单），通过要求用户从特定选项中进行选择来限制回答选项。

**原因如下**

使用预定义字段，用户只需单击一个框或从菜单中选择即可，从而确保对该问题的所有答案都相同。

一致、准确的数据对于准确的报告至关重要。 不一致的数据导致报告不准确，这可能会影响个人层面的决策。 此外，这种一致的数据允许您向报表中添加图表，以便直观地表示您的数据。 无限制的文本字段不能在图表中使用。

</br>
</br>

**最佳实践**

确保字段标签措辞明确且具有描述性。

**原因如下**

这样，填写自定义表单的人员就知道需要什么信息。

</br>
</br>

**最佳实践**

在自定义字段的说明字段中添加信息，以指示填写该字段的人员必须输入哪些信息。

**原因如下**

当您将鼠标悬停在？ 自定义表单中字段旁边的图标。 包括应在字段中输入的数据，包括文本字段的任何所需格式。

为用户提供更多详细信息有助于减少其他对话、来回电子邮件或混淆。 当信息不完整或丢失时，可能会延迟完成的工作。

</br>
</br>

**最佳实践**

以特定方式填写其他字段时，使用显示逻辑显示所需字段。 在自定义表单中使用跳过逻辑可隐藏与请求类型无关的字段。

**原因如下**

仅显示所需的字段或隐藏不需要的字段，会创建更干净的自定义表单，并减少用户在填写自定义表单时的混淆。 这也有助于缩短整体时限，减少用户的恫吓作用，提高回应率。

使用显示逻辑还可以帮助您减少需要创建和维护的自定义表单的数量。

</br>
</br>

**最佳实践**

通过从其他表单中提取计算信息，减少自定义表单上所需的计算数量。

**原因如下**

例如，问题表单上有一个名为“资产数量”的计算字段，用于计算附加到项目的资产数量。 此参数用于请求队列。 在转换请求时，您需要将此信息转移到项目中。 制作问题表单的副本并将其另存为项目表单。 然后，将问题计算字段的名称添加到项目表单的计算框中。 在此示例中，在项目表单中名为“资产数量”的计算字段中，按字面含义输入“资产数量”。 这会阻止Workfront尝试在项目上重新运行此计算，而是使用问题自定义表单中的值。

</br>
</br>

**最佳实践**

适用时，使用字段库中用途相同的现有字段。

**原因如下**

Workfront中的两个字段不能具有相同的名称。 如果字段具有通用名称，则该字段可能已存在。 创建新字段之前检查字段库，以查看字段是否已存在。

使用未创建的字段时，请记住，对此字段的修改会影响该字段所属的所有自定义表单。 如果需要更改计算或更改字段类型（例如，将文本更改为单选按钮），则需要创建一个新字段，并为其指定与原始字段不同的名称。 （请记住，名称相似的多个字段可能会让用户感到困惑。）

</br>
</br>

**最佳实践**

使用必填字段确保捕获到对您的流程或组织至关重要的信息。

**原因如下**

自定义表单上的数据不完整可能会延迟工作并影响报表。 必填字段指示器（字段名称旁边的红色*）提醒用户，在编辑然后保存该自定义表单或正式提交请求之前需要特定信息。

但是，应谨慎谨慎使用必填字段。 要求填写每个字段可能会阻止用户填写有用且完整的信息。 此外，在对象的“详细信息”区域中编辑自定义表单时，必填字段不完整会导致自定义表单无法保存。

</br>
</br>

**最佳实践**

在自定义表单中更改字段名称时请务必谨慎，因为这可能会影响调用该字段的计算字段。

**原因如下**

更改字段名称时，需要更新该名称，以在自定义表单的计算自定义字段中或内置到文本模式的计算中使用它。 更改字段名称可能会中断计算并导致信息不准确。

</br>
</br>

**最佳实践**

作为常规Workfront系统维护的一部分，定期查看自定义表单和字段，例如每季度一次。

**原因如下**

如果自定义表单和字段未收集您的组织完成工作所需的数据，则它们没有用处。

在进行更新时，请注意这些更改会如何影响Workfront的其他方面。 例如，更改字段名称可能会中断使用该字段的计算。 或者更改表单的标签或名称可能会导致报表不显示所需信息，或者导致与其他系统的集成无法运行。

除了更新自定义表单之外，请识别使用率低或根本未使用的表单。 未使用的表单能否频繁与其他表单组合？ 也可能是时候弃用该表单了，因为您的团队不再需要表单正在收集的信息。

在对这些不必要的表单执行任何操作之前，请参阅停用与删除自定义表单和自定义字段。

</br>
</br>

**最佳实践**

停用不需要的自定义表单，而不是删除它们。

**原因如下**

当您删除自定义表单时，将删除通过该自定义表单输入的所有自定义数据。
停用自定义表单会保留所有关联的历史数据。 这意味着您可以继续报告此信息。

停用还会阻止表单显示在用户选择自定义表单的下拉菜单中。 但是，表单仍会显示在它已经附加到的任何对象上。

</br>
</br>

**最佳实践**

在分区界限下隐藏自定义表单上不再需要的自定义字段。 然后，使此部分仅对系统管理员可见。

**原因如下**

删除现有自定义字段不仅会将其从自定义表单中删除，而且会删除该字段中包含的所有数据（无论该自定义字段是否使用）。 这意味着删除报表可能需要的历史数据。

要防止数据丢失，请在自定义表单本身中隐藏不需要的字段。 通过自定义表单中的分区界限，您可以根据用户是否有权查看、参与或管理其附加到的Workfront对象，隐藏或公开属于该分区一部分的字段。 或者，也可以将部分设置为“仅管理员”，因此只有具有系统管理员访问权限级别的用户才能查看表单的整个部分。

</br>
</br>

**最佳实践**

限制可以在Workfront实例中创建自定义表单的人员。


**原因如下**

让一组选定的人员来创建自定义表单，有助于控制在Workfront实例中创建的自定义表单的数量。

此外，允许其他人创建表单可以减轻系统管理员的工作量，并让每个组都能控制他们定期使用的自定义表单的更新。
