# 最佳实践 — 自定义表单

## 什么是Adobe Workfront“最佳实践”？

最佳做法是代表有效、高效的行动方针的准则；易于您和贵公司用户采用；和可以在您的组织内成功复制。

在您查看这些建议时，请记住，一些Workfront最佳实践是普遍的，而其他一些则可能更具体地介绍该主题。 将这些最佳实践用作框架，以帮助指导Workfront系统的设置和使用。

## 导航此页面

在您浏览此页面时，首先您会找到该主题所有最佳实践的高级列表。 这样，您便可以审核推荐，而无需深入研究“原因”的详细信息。

“为什么这些最佳实践？” 高级列表后找到的区域，详细介绍了一些最佳实践以及它们被视为流程、工具等的原因，您应当考虑使用Workfront实例进行实施。

</br>
</br>

## 自定义表单最佳实践

* 在Adobe Workfront中创建自定义表单之前，先在白板或纸上绘制其图表。

* 如果使用该模板创建的项目始终需要附加某些自定义表单，请将自定义表单附加到项目模板。 您甚至可以填写很少更改或需要特定信息的字段。

* 创建自定义表单以跟踪有关作为公司添加到Workfront系统的外部客户或供应商的信息。

* 创建一个包含常用字段和计算字段的“常规”或“通用”自定义表单，而不是将各个字段跟踪到多个表单中。 这有助于您以一致的方式收集企业范围的信息。

* 向自定义表单中添加部分中断，以使其保持有序且易于理解。

* 缩短自定义表单，以确保表单已完全填写，并且您获得了所需的所有信息。

* 使用预定义的自定义字段选项（如单选按钮和下拉菜单），通过要求用户从特定选项中进行选择来限制答案选项。

* 确保字段标签的措辞清晰且具有描述性。

* 在“说明”字段中添加有关自定义字段的信息，以指明填写该字段的人员必须输入哪些信息。

* 当以特定方式填写其他字段时，使用显示逻辑显示所需的字段。 在自定义表单中使用跳过逻辑来隐藏与请求类型无关的字段。

* 通过从其他表单中提取计算信息，减少自定义表单上所需的计算数量。

* 在为自定义表单创建新字段之前，请检查字段库，以查看该字段是否已存在。

* 使用必填字段可确保捕获对您的流程或组织至关重要的信息。

* 在自定义表单上更改字段名称时请务必小心，因为这会影响调用该字段的计算字段。

* 定期查看自定义表单和字段（例如每季度一次），作为常规Workfront系统维护的一部分。

* 停用不需要的自定义表单，而不是删除它们。

* 在分段下隐藏自定义表单上不再需要的自定义字段。 然后，使该部分仅对系统管理员可见。

* 限制在Workfront实例中创建自定义表单的人员。

## 为什么是这些最佳实践？

**最佳实践**

在Adobe Workfront中创建自定义表单之前，先在白板或纸上绘制其图表。

**原因如下**

确保您有用于填写所有必需信息的字段，并且这些字段的组织方式将便于用户填写表单。 对表单进行图示还可以帮助您确定是否可以使用显示逻辑隐藏/显示某些字段。

</br>
</br>

**最佳实践**

如果使用该模板创建的项目始终需要附加某些自定义表单，请将自定义表单附加到项目模板。 您甚至可以填写很少更改或需要特定信息的字段。

**原因如下**

这样，表单即已附加，部分信息已经填写完，这有助于加快项目创建速度，并确保所有适用项目中正确完整地填写这些自定义字段。

</br>
</br>

**最佳实践**

创建自定义表单以跟踪有关作为公司添加到Workfront系统的外部客户或供应商的信息。

**原因如下**

使用自定义表单跟踪地址、主要联系人的姓名等，以便在Workfront中轻松访问。 您还可以将此自定义表单信息提取到报表中。

</br>
</br>

**最佳实践**

创建一个包含常用字段和计算字段的“常规”或“通用”自定义表单，而不是将各个字段跟踪到多个表单中。 这有助于您以一致的方式收集企业范围的信息。

**原因如下**

“常规”表单使维护这些字段变得更加容易，因为它们都位于一个位置。 您将能够更新单个表单，而不是让这些单独的字段位于多个不同的表单上，然后您必须单独更新这些表单。

</br>
</br>

**最佳实践**

向自定义表单中添加部分中断，以使其保持有序且易于理解。

**原因如下**

将相关信息分组到各个部分有助于用户导航表单。

</br>
</br>

**最佳实践**

缩短自定义表单，以确保表单已完全填写，并且您获得了所需的所有信息。

**原因如下**

长表单对用户而言可能会令人生畏，并且通常会导致表单未完全填写。 这反过来又会导致分配信息不完整和报告数据不准确。

如果自定义表单具有许多字段，请尽可能并排放置相关字段，这样用户便无需进行滚动。 您还可以使用跳过逻辑来隐藏不需要填写的字段，或显示逻辑来显示特定字段。

</br>
</br>

**最佳实践**

使用预定义的自定义字段选项（如单选按钮和下拉菜单），通过要求用户从特定选项中进行选择来限制答案选项。

**原因如下**

对于预定义字段，用户只需单击框或从菜单中进行选择，即可确保该问题的所有答案都相同。

一致、准确的数据对于准确报告至关重要。 数据不一致会导致报表不准确，这可能会从单个级别上影响决策。 此外，此一致的数据允许您向报表中添加图表，以便直观地表示数据。 不能在图表中使用不限长度的文本字段。

</br>
</br>

**最佳实践**

确保字段标签的措辞清晰且具有描述性。

**原因如下**

这样，填写自定义表单的人员便能够了解需要哪些信息。

</br>
</br>

**最佳实践**

在“说明”字段中添加有关自定义字段的信息，以指明填写该字段的人员必须输入哪些信息。

**原因如下**

当您将鼠标悬停在？ 自定义表单中字段旁边的图标。 包括应在字段中输入的数据，包括文本字段的任何必需格式。

为用户提供更多详细信息有助于减少其他对话、往来电子邮件或混淆。 当信息不完整或缺失时，可能会延迟完成工作。

</br>
</br>

**最佳实践**

当以特定方式填写其他字段时，使用显示逻辑显示所需的字段。 在自定义表单中使用跳过逻辑来隐藏与请求类型无关的字段。

**原因如下**

仅显示所需字段或隐藏不需要的字段，可创建更简洁的自定义表单，并减少用户在填写自定义表单时的混淆。 这也有助于整体更短的形式，这样就不会令用户感到害怕，并促进更高的响应率。

使用显示逻辑还可以帮助您减少创建和维护自定义表单所需的数量。

</br>
</br>

**最佳实践**

通过从其他表单中提取计算信息，减少自定义表单上所需的计算数量。

**原因如下**

例如，在问题表单上有一个名为“资产数量”的计算字段，用于计算附加到该项目的资产数量。 请求队列中使用此插件。 在请求转换后，您需要此信息才能将其传递到项目。 制作问题表单的副本，并将其另存为项目表单。 然后，将问题计算字段的名称添加到项目表单的计算框中。 在此示例中，在项目表单名为“资产数量”的计算字段中，字面输入“资产数量”。 这可以防止Workfront尝试在项目上重新运行此计算，而是使用问题自定义表单中的值。

</br>
</br>

**最佳实践**

在适用时，使用字段库中具有相同用途的现有字段。

**原因如下**

Workfront中的两个字段不能具有相同的名称。 如果字段具有通用名称，则该字段可能已存在。 在创建新字段之前，请检查字段库以查看该字段是否已存在。

使用未创建的字段时，请记住，对此字段的修改会影响字段所属的所有自定义表单。 如果您需要更改计算或更改字段类型（例如，将文本更改为单选按钮），则需要创建一个新字段，并为其指定与原始字段不同的名称。 （请记住，名称相似的多个字段可能会令用户感到困惑。）

</br>
</br>

**最佳实践**

使用必填字段可确保捕获对您的流程或组织至关重要的信息。

**原因如下**

自定义表单上的数据不完整，可能会延迟工作并影响报表。 必填字段指示符（字段名称旁边的红色*）会提醒用户，在编辑并保存该自定义表单或正式提交请求之前，需要提供特定信息。

但是，必填字段应谨慎使用。 需要填写每个字段可能会阻止用户填写包含有用且完整信息的字段。 此外，在对象的“详细信息”区域中编辑自定义表单时，不完整的必填字段会阻止保存自定义表单。

</br>
</br>

**最佳实践**

在自定义表单上更改字段名称时请务必小心，因为这会影响调用该字段的计算字段。

**原因如下**

更改字段名称时，需要更新该名称，在自定义表单上的计算自定义字段或内置到文本模式的计算中使用该名称。 更改字段名称可能会中断计算，并导致信息不准确。

</br>
</br>

**最佳实践**

定期查看自定义表单和字段（例如每季度一次），作为常规Workfront系统维护的一部分。

**原因如下**

自定义表单和字段如果没有收集贵组织完成工作所需的数据，则它们将不会有用。

进行更新时，请注意这些更改对Workfront其他方面有何影响。 例如，更改字段名称可能会中断使用该字段的计算。 或者，更改表单的标签或名称可能会导致报表不显示所需信息，或与其他系统的集成无法运行。

除了更新自定义表单之外，还应识别使用率较低或根本未使用的表单。 不常用的表单能否与其他表单组合使用？ 或者，该弃用该表单了，因为您的团队不再需要该表单收集的信息。

在对这些不必要的表单执行任何操作之前，请阅读有关停用和删除自定义表单和自定义字段的信息。

</br>
</br>

**最佳实践**

停用不需要的自定义表单，而不是删除它们。

**原因如下**

删除自定义表单时，会删除通过该自定义表单输入的所有自定义数据。
取消激活自定义表单会保留所有关联的历史数据。 这意味着您可以继续报告此信息。

取消激活还会阻止表单显示在用户选择自定义表单的下拉菜单中。 但是，表单仍会显示在已附加到的任何对象上。

</br>
</br>

**最佳实践**

在分段下隐藏自定义表单上不再需要的自定义字段。 然后，使该部分仅对系统管理员可见。

**原因如下**

删除现有的自定义字段不仅会将其从自定义表单中删除，还会删除该字段中包含的所有数据（无论该自定义字段在何处使用）。 这意味着会删除报表可能需要的历史数据。

要防止数据丢失，请在自定义表单中隐藏不需要的字段。 通过自定义表单中的节中断，您可以根据用户是否有权查看、贡献或管理附加到的Workfront对象来隐藏或显示属于该节的字段。 或者，可以将某个部分设置为“仅管理员”，以便只有具有系统管理员访问权限级别的用户才能看到表单的整个部分。

</br>
</br>

**最佳实践**

限制在Workfront实例中创建自定义表单的人员。


**原因如下**

拥有一组可以创建自定义表单的选定人员，有助于控制在您的Workfront实例中创建的自定义表单数量。

此外，允许其他人创建表单可以减轻系统管理员的工作负担，并让每个组可以定期控制他们使用的自定义表单的更新。
