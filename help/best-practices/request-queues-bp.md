# 最佳实践 — 请求队列

## 什么是Adobe Workfront“最佳实践”？

最佳实践是指导准则，它们代表了一个有效、高效的操作过程；您和您公司的用户可轻松采用；并且可在您的整个组织中成功复制。

在查看这些建议时，请牢记，某些Workfront最佳实践是通用的，而其他最佳实践可能更特定于此主题。 使用这些最佳实践作为框架，帮助指导您的Workfront系统设置和使用。

## 导航此页面

滚动浏览本页时，首先您会找到该主题所有最佳实践的高级列表。 这样，您就可以查看推荐，而无需深入了解“原因”的详细信息。

“为什么这些是最佳实践？” 区域位于高级列表之后，它提供了一些最佳实践的更多详细信息，以及为什么将它们视为流程、工具等，您应考虑使用Workfront实例进行实施。

</br>
</br>

## 请求队列最佳实践

* 包括请求队列中每个元素的说明，这些元素包括请求队列项目、主题组、队列主题和路由规则。

* 创建一个名为“请求队列”或“正在运行”的项目状态（等同于“当前”），以将请求队列项目与其他项目区分开。

* 如果您计划将问题审批与通过队列提交的请求结合使用，请创建一个名为“已拒绝”的问题状态。

* 将“通用”自定义表单分配给请求队列，以捕获尽可能多的企业范围的一致数据。

* 避免与“每个人”共享请求队列。 设置队列详细信息设置，以便用户只能看到与其需求相关的队列。

* 构建并分配包含请求队列报告的功能板，以便流量管理员、系统管理员或分配的用户可以直接处理问题。

* 使用布局模板从项目的左侧面板菜单中移除请求队列设置选项，以供不需要创建队列的用户使用。

* 创建系统管理员请求队列，以便用户提问Workfront相关问题、提出有关系统设置的请求、安排新用户培训等。

* 审核请求队列会定期识别，并取消共享未使用的队列。

* 使用主题组在请求队列中整理10个以上的队列主题，以创建更短、更易于管理的列表。

* 通过使用主题组和队列主题而不是创建多个队列来划分请求队列，控制用户可用的请求队列总数。

* 为每个队列主题设置路由规则。 至少应设置一个默认路由规则。

* 当需要选择路由时，请利用主题组和队列主题。

* 将请求路由到团队而不是个人。

## 为什么有这些最佳实践？


**最佳实践**

包括请求队列中每个元素的说明，这些元素包括请求队列项目、主题组、队列主题和路由规则。

**原因如下**

通过描述，组管理员、未来的系统管理员或维护请求队列的其他人员可以准确地了解请求队列中每个部分的用途。

当您将鼠标悬停在新请求窗口中的字段上的信息图标上时，也会显示说明信息。

描述不一定很长，只需简要评论一下元素的用途或用途。

</br>
</br>

**最佳实践**

创建一个名为“请求队列”或“正在运行”的项目状态（等同于“当前”），以将请求队列项目与其他项目区分开。

**原因如下**

请求队列在项目中一直存在，并且状态必须等于当前才能使队列处于活动状态。

为了将请求与具有“当前”状态的实际工作项目区分开来，请创建一个仅用于名为“请求队列”或“可操作”的请求队列的状态。 然后，您可以使用此状态来帮助在编写报告时排除或包含请求队列项目。

</br>
</br>

**最佳实践**

使用问题批准时创建名为“已拒绝”的问题状态，并将“如果被拒绝”选项设置为“已拒绝”状态。

**原因如下**

通过使用“已拒绝”状态，可以明确表示请求已审核并已拒绝。

</br>
</br>

**最佳实践**

将“通用”自定义表单分配给请求队列，以捕获尽可能多的企业范围的一致数据。

**原因如下**

“通用”自定义表单可收集请求所需的标准信息，无论提交的请求类型如何。

拥有“通用”自定义表单有助于减少您需要创建和维护的自定义表单的数量。 它还可确保所有请求都以相同的方式收集相同的信息，从而实现一致的报表和数据分析。

</br>
</br>

**最佳实践**

避免与“每个人”共享请求队列。  设置队列详细信息设置，以便用户只能看到与其需求相关的队列。

**原因如下**

在大多数情况下，请求队列只需要与特定一组人员（如团队、供应商、客户等）共享。 当请求者在请求队列列表中仅看到他们需要的内容时，可轻松地查找和导航内容。

</br>
</br>


**最佳实践**

构建并分配包含请求队列报告的功能板，以便流量管理员、系统管理员或分配的用户可以直接处理问题。

**原因如下**

让用户能够快速轻松地访问传入请求，意味着工作不会因洗牌而丢失。

</br>
</br>

**最佳实践**

使用布局模板从项目的左侧面板菜单中移除请求队列设置选项，以供不需要创建队列的用户使用。


**原因如下**

这可确保所有请求队列都经过适当的创建过程（例如由治理委员会审核），并由系统管理员或组管理员正确设置。

此外，这有助于使队列列表保持井然有序，并重点关注组织所需的请求类型。

</br>
</br>

**最佳实践**

创建系统管理员请求队列，以便用户提问Workfront相关问题、提出有关系统设置的请求、安排新用户培训等。

**原因如下**

这为用户提交问题以及管理员收集、监控和响应Workfront相关问题提供了一个集中的位置。

此外，此信息还可用于向领导层展示系统管理员角色所花费的时间、精力和价值，并作为添加系统管理员的理由。

</br>
</br>


**最佳实践**

审核请求队列会定期识别，并取消共享未使用的队列。

**原因如下**

定期审核Adobe Workfront系统中的设置和项目，有助于保持系统整洁且没有不需要的项目。 如果队列不再被使用或监视，请确保用户不能再访问它，以免工作请求成为空白。

</br>
</br>


**最佳实践**

使用主题组在请求队列中整理10个以上的队列主题，以创建更短、更易于管理的列表。

**原因如下**

主题组通过减少可选择的初始选项列表而增加用户采用率并减少混淆。 这使用户能够轻松地找到他们要查找的内容，而不会在尝试提交请求时令他们不知所措。

此外，它使系统管理员和/或请求队列管理员有机会为用户创建平滑的导航路径，并更好地组织和报告提交的请求类型。

</br>
</br>

**最佳实践**

通过使用主题组和队列主题而不是创建多个队列来划分请求队列，控制用户可用的请求队列总数。

**原因如下**

请求队列过多，用户很难找到所需的内容。

更少的队列还可以帮助流量协调员、系统管理员或其他管理队列的人员，使他们能够更快地找到所需的信息，而无需导航到多个请求队列项目。

如果不同的请求队列需要不同的访问权限，或者如果合并队列会给用户带来混淆，请创建多个请求队列。

</br>
</br>

**最佳实践**

为每个队列主题设置路由规则。 至少应设置一个默认路由规则。

**原因如下**

路由规则可确保始终为某人分配传入请求，以便工作不会受到影响。

</br>
</br>

**最佳实践**

当需要选择路由时，请利用主题组和队列主题。

**原因如下**

路由规则无法应用于自定义表单的字段。 因此，如果需要将不同类型的请求路由到不同的团队/个人，则请使每种类型的请求都有自己的主题组/队列主题，以便能够正确路由工作。

</br>
</br>

**最佳实践**

将请求路由到团队而不是个人。

**原因如下**

当请求被发送到团队时，它使整个团队能够了解正在提出的请求以及即将开展的工作可能涉及的内容。 每个人都可以查看“团队”页面上的新项目或跟踪功能板上的报告的新增功能。

它还确保当流量管理员或负责审阅或分配传入请求的其他人不可用时，备份会自动可用，并且有权访问请求信息。
