# 最佳实践 — 请求队列

## 什么是Adobe Workfront“最佳实践”？

最佳做法是代表有效、高效的行动方针的准则；易于您和贵公司用户采用；和可以在您的组织内成功复制。

在您查看这些建议时，请记住，一些Workfront最佳实践是普遍的，而其他一些则可能更具体地介绍该主题。 将这些最佳实践用作框架，以帮助指导Workfront系统的设置和使用。

## 导航此页面

在您浏览此页面时，首先您会找到该主题所有最佳实践的高级列表。 这样，您便可以审核推荐，而无需深入研究“原因”的详细信息。

“为什么这些最佳实践？” 高级列表后找到的区域，详细介绍了一些最佳实践以及它们被视为流程、工具等的原因，您应当考虑使用Workfront实例进行实施。

</br>
</br>

## 请求队列最佳实践

* 包括请求队列的每个元素的描述 — 请求队列项目、主题组、队列主题和路由规则。

* 创建一个名为“请求队列”或“运行”的项目状态，该状态等于“当前”，以便将请求队列项目与其他项目区分开。

* 如果您计划对通过队列提交的请求使用问题批准，请创建一个名为“已拒绝”的问题状态。

* 为请求队列分配“通用”自定义表单，以捕获尽可能多的企业范围一致数据。

* 避免与“每个人”共享请求队列。 设置队列详细信息设置，以便用户只能查看与其需求相关的队列。

* 生成并分配包含请求队列报表的功能板，以便流量管理器、系统管理员或分配的用户可以直接处理问题。

* 使用布局模板从项目的左侧面板菜单中为不需要创建队列的用户删除请求队列设置选项。

* 创建系统管理员请求队列，让用户提出与Workfront相关的问题、发出有关系统设置的请求、计划新的用户培训等。

* 定期审核请求队列，以识别和取消共享未使用的队列。

* 使用主题组可组织请求队列中10多个队列主题，以创建更短、更易于管理的列表。

* 通过使用主题组和队列主题划分请求队列，而不是创建多个队列，可控制用户可用的请求队列总数。

* 为每个队列主题设置路由规则。 至少应设置默认路由规则。

* 在需要选择路由时利用主题组和队列主题。

* 将请求路由到团队，而不是个人。

## 为什么是这些最佳实践？


**最佳实践**

包括请求队列的每个元素的描述 — 请求队列项目、主题组、队列主题和路由规则。

**原因如下**

通过描述，组管理员、将来的系统管理员或维护请求队列的其他人员可以准确了解每个请求队列的用途。

当您将鼠标悬停在新请求窗口中字段上的信息图标上时，也会显示描述信息。

描述不必太长，只是有关元素用途或用法的简短评论。

</br>
</br>

**最佳实践**

创建一个名为“请求队列”或“运行”的项目状态，该状态等于“当前”，以便将请求队列项目与其他项目区分开。

**原因如下**

请求队列在项目中“处于活动状态”，且状态必须等于“当前”，队列才能处于活动状态。

为了将请求与具有“当前”状态的实际工作项目区分开来，请创建一个状态，该状态将仅在名为“请求队列”或“操作”的请求队列上使用。 然后，您可以使用此状态来帮助在编写报表时排除或包含请求队列项目。

</br>
</br>

**最佳实践**

使用问题批准时，创建一个名为“已拒绝”的问题状态，并将“如果已拒绝”选项设置为“已拒绝”状态。

**原因如下**

通过使用“已拒绝”状态，这可以明确表示请求已被审核和拒绝。

</br>
</br>

**最佳实践**

为请求队列分配“通用”自定义表单，以捕获尽可能多的企业范围一致数据。

**原因如下**

“通用”自定义表单可收集请求所需的标准信息，而无论提交的请求类型如何。

拥有“通用”自定义表单有助于减少您创建和维护自定义表单所需的数量。 它还可确保所有请求以相同的方式收集相同的信息，从而实现一致的报告和数据分析。

</br>
</br>

**最佳实践**

避免与“每个人”共享请求队列。  设置队列详细信息设置，以便用户只能查看与其需求相关的队列。

**原因如下**

在大多数情况下，请求队列只需与一组特定人员（如团队、供应商、客户等）共享。 当请求者在请求队列列表中只看到他们需要的内容时，可以轻松查找和导航。

</br>
</br>


**最佳实践**

生成并分配包含请求队列报表的功能板，以便流量管理器、系统管理员或分配的用户可以直接处理问题。

**原因如下**

让用户快速、轻松地访问传入的请求，意味着工作不会在混洗中丢失。

</br>
</br>

**最佳实践**

使用布局模板从项目的左侧面板菜单中为不需要创建队列的用户删除请求队列设置选项。


**原因如下**

这可确保所有请求队列都经过适当的创建过程（例如，由管理委员会审核），并由系统或组管理员正确设置。

此外，这有助于保持队列列表的有序性，并重点关注您的组织需要的请求类型。

</br>
</br>

**最佳实践**

创建系统管理员请求队列，让用户提出与Workfront相关的问题、发出有关系统设置的请求、计划新的用户培训等。

**原因如下**

这为用户提交问题提供了一个集中位置，并为管理员收集、监控和响应与Workfront相关的问题。

此外，此信息还可用于显示系统管理员角色的领导时间、工作和价值，并作为其他系统管理员的理由。

</br>
</br>


**最佳实践**

定期审核请求队列，以识别和取消共享未使用的队列。

**原因如下**

对Adobe Workfront系统中的设置和项目进行定期审核，有助于使其保持整洁，并避免不需要的项目。 如果队列不再被使用或监视，请确保用户无法再访问该队列，以便工作请求不会陷入空白。

</br>
</br>


**最佳实践**

使用主题组可组织请求队列中10多个队列主题，以创建更短、更易于管理的列表。

**原因如下**

主题组通过减少要从中选择的选项的初始列表来增加用户采用，并减少混淆。 这样，用户便可以轻松找到自己想要的内容，而无需在尝试提交请求时压倒自己。

此外，它还允许系统管理员和/或请求队列管理器有机会为用户创建一个流畅的导航路径，以及更好的方式以便组织和报告正在提交的请求类型。

</br>
</br>

**最佳实践**

通过使用主题组和队列主题划分请求队列，而不是创建多个队列，可控制用户可用的请求队列总数。

**原因如下**

过多的请求队列使用户很难找到所需内容。

减少队列也有助于流量协调员、系统管理员或其他人管理队列，使他们能够更快地找到所需的信息，而无需导航到多个请求队列项目。

如果不同请求队列需要不同的访问，或者合并队列会令用户感到困惑，则创建多个请求队列。

</br>
</br>

**最佳实践**

为每个队列主题设置路由规则。 至少应设置默认路由规则。

**原因如下**

路由规则可确保将始终为某人分配传入的请求，以便工作不会漏掉。

</br>
</br>

**最佳实践**

在需要选择路由时利用主题组和队列主题。

**原因如下**

路由规则不能应用于自定义表单的字段。 因此，如果需要将不同类型的请求路由到不同的团队/个人，则请使每种类型的请求具有其自己的主题组/队列主题，以便能够正确路由工作。

</br>
</br>

**最佳实践**

将请求路由到团队，而不是个人。

**原因如下**

在向团队发送请求时，它可以让整个团队了解正在发出的请求以及即将开展的工作可能需要哪些内容。 每个人都可以在“团队”页面中查看新项目，或通过功能板上的报表来跟踪新增功能。

它还可确保当流量管理员或负责审核或分配传入请求的其他人员不可用时，备份会自动可用，并有权访问请求信息。
