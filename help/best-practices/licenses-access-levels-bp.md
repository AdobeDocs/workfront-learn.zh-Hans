# 最佳实践 — 许可证和访问级别

## 什么是Adobe Workfront“最佳实践”？

最佳做法是代表有效、高效的行动方针的准则；易于您和贵公司用户采用；和可以在您的组织内成功复制。

在您查看这些建议时，请记住，一些Workfront最佳实践是普遍的，而其他一些则可能更具体地介绍该主题。 将这些最佳实践用作框架，以帮助指导Workfront系统的设置和使用。

## 导航此页面

在您浏览此页面时，首先您会找到该主题所有最佳实践的高级列表。 这样，您便可以审核推荐，而无需深入研究“原因”的详细信息。

“为什么这些最佳实践？” 高级列表后找到的区域，详细介绍了一些最佳实践以及它们被视为流程、工具等的原因，您应当考虑使用Workfront实例进行实施。

</br>
</br>

## 许可证和访问级别最佳实践

* 在设置访问级别时，首先要减少用户的访问权限。

* 分配审核和请求许可证时，通常默认使用审核，因为它为用户提供了更多的Adobe Workfront权限。

* 在所有访问级别中取消选中每个对象上的“共享系统范围”复选框，除非您特别希望这些用户能够执行此操作。

* 请考虑在访问级别设置其他限制下激活“绝不允许用户删除评论”设置。

* 限制系统管理员的数量，以支持组管理员。

* 复制现有访问级别并进行修改，而不是从头开始创建新访问级别。

* 记录每个访问级别在描述框中可以执行的操作。

* 将您自己限制为仅满足完成工作目标所必需的访问级别，最好是满足系统中大多数用户需求的4或5级。

* 为至少两个用户分配全局系统管理员访问级别。

* 限制用户通过共享对Workfront项目执行的操作，而不是删除访问级别中的功能。

</br>
</br>


## 为什么是这些最佳实践？

**最佳实践**

在设置访问级别时，首先要减少用户的访问权限。



**原因如下**

以用户完成工作所需的最低访问权限开始用户。 如果由于访问权限不足而无法执行作业，则通常会请求附加访问权限。 立即授予用户过多访问权限可能会导致安全问题。 此外，向用户提供更多访问权限总比取消访问权限更好。

</br>
</br>



**最佳实践**

分配审核和请求许可证时，通常默认使用审核，因为它为用户提供了更多的Adobe Workfront权限。



**原因如下**

尽管可以将审核许可证和请求许可证分配给Workfront中的无限数量的用户，但请求许可证仅限于几乎仅发出和更新请求。 与“请求”许可证相比，“审阅”许可证对项目和任务的访问权限更多，而且还可以查看项目组合和项目、编辑文档和访问资源管理工具。

</br>
</br>

**最佳实践**

在所有访问级别中取消选中每个对象上的“共享系统范围”复选框，除非有特定原因需要用户能够共享系统范围。



**原因如下**

在系统范围内共享对象通常用作拐杖，以允许某些用户在Workfront中查看项目。 当Workfront组结构缺失或共享权限未完全理解时，会发生这种情况。 在系统范围内共享项目时，这意味着每个人都可以看到已共享的项目。 根据系统中保留的信息类型，这可能会导致隐私问题。



例如，您可能正在与Workfront内的多家供应商合作，以检查进度、提供批准等。 如果“共享系统范围”复选框是一个选项，则可以选中该选项或将其设置为默认选项，以便所有供应商都能查看该选项。



通过完全取消选中该选项，您可以让具有共享权限的用户确定要与其共享对象的特定人员或人员（通过公司、组或团队）。

</br>
</br>

**最佳实践**

请考虑在访问级别设置其他限制下激活“绝不允许用户删除评论”设置。



**原因如下**

激活此选项可确保不会从Workfront中删除过去的通信。 有些组织要求保留完整的评论历史记录以便进行审核。

</br>
</br>

**最佳实践**

限制系统管理员的数量，以支持组管理员。



**原因如下**

系统管理员有权访问Workfront中的所有内容，包括全局系统设置。 设置组管理员可以访问的权限由系统管理员控制，并且只能应用于该特定组。



通过让组管理员，系统管理员可以委派许多责任，从而让他们能够专注于大图景项目，而不是日常维护Workfront。 群组管理员可以更轻松地与群组的需求保持联系，从而为用户提供更好的服务。

</br>
</br>


**最佳实践**

复制现有访问级别并进行修改，而不是从头开始创建新访问级别。



**原因如下**

复制现有访问级别为新访问级别提供了一致的基础，可确保初始设置相同。 这也节省了时间，因为系统管理员不必完全从头开始设置访问级别。

</br>
</br>

**最佳实践**

记录每个访问级别在描述框中可以执行的操作。



**原因如下**

详细说明，列出每种对象类型的设置。 这有助于系统管理员（现在和将来）准确了解每个访问级别的功能，而无需深入研究访问级别本身来检查设置。



这样，在报表中查看访问级别时，也可以更轻松地比较访问级别。 可以快速将描述字段添加到视图中，以快速了解它们之间的差异，以及创建不同访问级别的可能原因。

</br>
</br>

**最佳实践**

将您自己限制为仅满足完成工作目标所必需的访问级别，最好是满足系统中大多数用户需求的4或5级。


**原因如下**

访问级别可确保在与用户共享Workfront对象时，用户拥有编辑、删除对象等所需的权限。 您可以使访问级别更一般，因为可以将在单个项目上共享的内容配置为更加具体。


此外，访问级别越低，维护无杂乱的系统和实施策略就越容易，这也可以在用户加入公司或更换部门时加快入门速度。

</br>
</br>

**最佳实践**

为至少两个用户分配全局系统管理员访问级别。

**原因如下**

多个人应该了解为何要以原样配置Workfront，如何管理/维护它，以及如何支持用户。 如果一个人不在办公室、离开组织、忙碌等，这可确保有另一个人拥有成功管理系统的信息和知识。

</br>
</br>

**最佳实践**

限制用户通过共享对Workfront项目执行的操作，而不是删除访问级别中的功能。


**原因如下**

访问级别控制用户可以在全局级别上对特定项目执行的操作。 每个项目、任务、组合、文档等的共享权限，可控制单个用户可以对该特定项目执行的操作。 微调特定项目的共享权限，而不是为具有特定访问级别的每个人删除功能，以便用户拥有有限的控制权。
