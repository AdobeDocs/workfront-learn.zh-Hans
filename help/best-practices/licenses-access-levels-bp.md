# 最佳实践 — 许可证和访问级别

## 什么是Adobe Workfront“最佳实践”？

最佳实践是指导准则，它们代表了一个有效、高效的操作过程；您和您公司的用户可轻松采用；并且可在您的整个组织中成功复制。

在查看这些建议时，请牢记，某些Workfront最佳实践是通用的，而其他最佳实践可能更特定于此主题。 使用这些最佳实践作为框架，帮助指导您的Workfront系统设置和使用。

## 导航此页面

滚动浏览本页时，首先您会找到该主题所有最佳实践的高级列表。 这样，您就可以查看推荐，而无需深入了解“原因”的详细信息。

“为什么这些是最佳实践？” 区域位于高级列表之后，它提供了一些最佳实践的更多详细信息，以及为什么将它们视为流程、工具等，您应考虑使用Workfront实例进行实施。

</br>

## 许可证和访问级别最佳实践

* 在设置访问级别时，首先减少用户的访问权限。

* 在分配“审阅”和“请求”许可证时，通常默认为“审阅”，因为它在Adobe Workfront中为用户提供更多权限。

* 取消选中所有访问级别中每个对象上的“共享系统范围”复选框，除非您明确希望这些用户能够执行此操作。

* 考虑激活访问级别中的设置其他限制下的“从不允许用户删除评论”设置。

* 限制系统管理员的数量，以利于组管理员。

* 复制现有访问级别并进行修改，而不是从头开始创建新的访问级别。

* 在“描述”框中记录每个访问级别可以执行的操作。

* 将您自己限制为仅提供实现工作目标所需的访问级别，最好是四个或五个级别，可满足系统中大多数用户的需求。

* 至少分配两个用户全局系统管理员访问级别。

* 限制用户可以通过共享来处理Workfront项目，而不是删除访问级别中的功能。

</br>
</br>


## 为什么有这些最佳实践？

**最佳实践**

在设置访问级别时，首先减少用户的访问权限。



**原因如下**

以用户完成工作所需的最低访问权开始用户。 如果由于访问权限不足而无法执行工作，他们通常会请求额外的访问权限。 立即授予用户过多的访问权限可能会导致安全问题。 此外，授予用户更多访问权限总比取消访问权限要好。

</br>
</br>



**最佳实践**

在分配“审阅”和“请求”许可证时，通常默认为“审阅”，因为它在Adobe Workfront中为用户提供更多权限。



**原因如下**

虽然Workfront中的审阅和请求许可证都可以分配给无限数量的用户，但请求许可证限制为几乎只能发出和更新请求。 与请求许可证相比，审阅许可证具有对项目和任务的更多访问权限，并且还可查看项目组合和程序、编辑文档以及访问资源管理工具。

</br>
</br>

**最佳实践**

取消选中所有访问级别中每个对象上的“共享系统范围”复选框，除非出于特定原因，用户需要能够共享系统范围。



**原因如下**

在系统范围内共享对象通常用作拐点，以允许某些用户查看Workfront中的项目。 当缺少Workfront组结构或共享权限未完全了解时，会发生这种情况。 当项目在系统范围内共享时，这意味着每个人都可以看到共享的项目。 根据系统中保存的信息类型，这可能会导致隐私问题。



例如，您可能正在与Workfront中的多家供应商合作，以检查进度、提供审批等。 如果“共享系统范围”复选框是一个选项，则可以选中或将其设置为默认值，使所有供应商都可以使用该信息。



通过完全取消选中选项，您可以使具有共享权限的用户必须决定要与之共享对象的特定人员或人员（通过公司、组或团队）。

</br>
</br>

**最佳实践**

考虑激活访问级别中的设置其他限制下的“从不允许用户删除评论”设置。



**原因如下**

激活此选项可确保不会从Workfront中删除过去的通信。 某些组织要求保留完整的注释历史记录以进行审核。

</br>
</br>

**最佳实践**

限制系统管理员的数量，以利于组管理员。



**原因如下**

系统管理员有权访问Workfront中的所有内容，包括全局系统设置。 系统管理员可以访问的设置组由系统管理员控制，并且仅应用于该特定组。



拥有组管理员让系统管理员可以委派许多职责，让他们专注于更大的项目，而不是Workfront的日常维护。 组管理员可以更轻松地与组的需求保持联系，从而为用户提供更好的服务。

</br>
</br>


**最佳实践**

复制现有访问级别并进行修改，而不是从头开始创建新的访问级别。



**原因如下**

复制现有访问级别为新访问级别提供了一致的基础，从而确保初始设置相同。 这还可以节省时间，因为系统管理员不必从头开始完全设置访问级别。

</br>
</br>

**最佳实践**

在“描述”框中记录每个访问级别可以执行的操作。



**原因如下**

详细说明并列出每种对象类型的设置。 这有助于目前和未来的系统管理员确切了解每个访问级别的作用，而不必深入了解访问级别本身以查看设置。



这还可以使在报告中查看访问级别时比较访问级别更容易。 可以快速将描述字段添加到视图中，以快速查看它们之间的差异以及可能创建不同访问级别的原因。

</br>
</br>

**最佳实践**

将您自己限制为仅提供实现工作目标所需的访问级别，最好是四个或五个级别，可满足系统中大多数用户的需求。


**原因如下**

访问级别可确保在与用户共享Workfront对象时，用户拥有编辑、删除它等所需的权限。 您可以使访问级别更通用，因为可以将单个项目上的共享配置为更特定。


此外，更少的访问级别可以更轻松地维护一个没有凌乱的系统并实施策略，这还可以在用户加入公司或切换部门时更快地上手。

</br>
</br>

**最佳实践**

至少分配两个用户全局系统管理员访问级别。

**原因如下**

多个人员应该了解为什么Workfront按原样进行配置、如何管理/维护以及如何支持用户。 如果一个人在外办公、离开组织、忙碌等，这就保证了有另一个人有信息和知识可以成功的管理这个系统。

</br>
</br>

**最佳实践**

限制用户可以通过共享来处理Workfront项目，而不是删除访问级别中的功能。


**原因如下**

访问级别控制用户可以对全局级别上的特定项目执行的操作。 每个项目、任务、项目组合、文档等的共享权限可控制单个用户可以对该特定项目执行的操作。 不要为具有特定访问级别的所有人删除功能，而是微调特定项目的共享权限，以便用户具有有限的控制权。
