# 浏览Workfront中的请求队列

此视频介绍Workfront中请求队列的概念、用途以及如何提交和管理请求。 请求&#x200B;队列用作提交各种类型请求的集中位置。 

>[!VIDEO](https://video.tv.adobe.com/v/335220/?quality=12&learn=on&enablevpops=1)

## 关键要点

* **请求队列的用途：** Workfront中的请求队列为提交各种类型的请求（如PTO、软件升级、客户端建议和培训需求）提供了一个集中的位置。
* **提交请求：**&#x200B;用户可以通过导航到“请求”区域，选择请求类型，填写“主题”、“描述”、“可选字段”（例如“文档”）等字段并完成自定义表单来提交请求。 &#x200B;
* **作为问题的请求：**&#x200B;在Workfront中，每个请求都被视为问题，必须关联到项目。 专门用于请求的&#x200B;项目称为请求队列。 
* **队列管理：**&#x200B;有效的队列管理涉及按类型组织请求，这会自定义所需信息和分配，从而提高效率。 &#x200B;
* **查找已提交的请求：**&#x200B;通过导航到“项目”区域并选择相关项目，可以在相关请求队列的“问题”选项卡中找到已提交的请求。 &#x200B;
