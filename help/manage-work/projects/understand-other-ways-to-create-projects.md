# 探索创建项目的四种方法

此视频提供了有关使用各种方法在Workfront中创建项目的全面指南：

**从模板创建项目：**

* 导航到项目页面，单击“新建项目”，然后选择“通过模板新建项目”。  
* 浏览活动模板、预览摘要（例如，持续时间、所有者、任务摘要、自定义表单）并选择模板。  
* 可直接访问收藏模板以加快选择速度。  
* 填写初始信息（例如项目名称），然后单击“创建项目”。 字段&#x200B;可在以后编辑。  

**从头开始创建项目：**

* 转到“项目”页面，单击“新建项目”，然后选择“新建项目”。  
* 命名项目并编辑共享选项、自定义数据、项目组合和项目群等详细信息。  
* 可以接受或更改组或系统管理员设置的默认值。  
* 保存项目，该项目最初仅与创建者和系统管理员共享。  
* 任务可通过键入名称进行添加，持续时间及计划小时数等字段可编辑。  

**正在复制项目：**

* 打开现有项目，单击三个圆点菜单，然后选择“复制”。  
* 更改名称、状态（如Planning），并取消选择不复制的项（如费用）。  
* 编辑详细信息，如复制后的计划开始日期。  

**正在导入Microsoft项目文件：**

* 将MS Project文件另存为XML文件。  
* 使用“新建项目”菜单中的“导入MS项目”选项，选择文件，然后开始导入。  
* 某些MS Project设置（例如，注释、自定义表单、里程碑）将不会传输。  
* 将状态设置为“正在计划”，以在导入的项目上线之前对其进行审查和编辑。  


此视频强调Workfront在通过模板、草稿、复制或导入来创建项目时为了满足不同用户需求的灵活性。

## 关键要点

* **模板简化了项目创建：**
* 强烈建议使用模板，因为它可以预先设置任务、表单和共享选项等关键详细信息，从而节省时间和精力。  
* **收藏的快速访问模板：**
* 收藏夹模板允许用户绕过选择过程并直接创建项目，从而简化工作流程。  
* **从头开始创建项目优惠灵活性：**
* 虽然更需要手动操作，但从头开始创建项目允许自定义所有详细信息，非常适合简单或专用项目。  
* **复制项目可节省时间：**
* 复制现有项目会复制其结构和设置，并使用选项来修改或排除特定元素。  
* **正在导入用于转换的MS Project文件：**
* Workfront支持导入MS Project文件（另存为XML），使用户能够转换正在进行的项目，但某些数据（例如，注释、里程碑）将不会传输。
